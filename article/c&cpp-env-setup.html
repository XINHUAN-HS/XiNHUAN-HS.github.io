⚙️ C/C++ 环境变量配置指南 — 2026-06-22

C/C++ 开发的第一步是正确配置编译器和构建工具的环境变量。本文涵盖 GCC、MinGW-w64 以及 Visual Studio 命令行工具的 PATH 设置，帮助你从终端直接调用编译器。

1. 编译器选择与环境变量概览

常见的 C/C++ 编译器包括：

  • GCC（Linux/macOS 默认）
  • MinGW-w64（Windows 下的 GCC 移植版）
  • MSVC（Visual Studio 自带编译器）

核心环境变量：

变量名 作用
PATH 添加编译器所在目录，使 gcc / g++ / cl 命令全局可用
CC 指定 C 编译器路径（常用于 Makefile）
CXX 指定 C++ 编译器路径
INCLUDE 头文件搜索路径（MSVC 常用）
LIB 库文件搜索路径（MSVC 常用）

2. Windows 下配置 MinGW-w64

2.1 下载与安装

访问 MinGW-w64 官网 下载安装包，或使用 niXman 的预编译版本。解压到 C:\mingw64。

2.2 配置 PATH

  1. 右键“此电脑” → 属性 → 高级系统设置 → 环境变量。
  2. 在“系统变量”中找到 Path，点击编辑。
  3. 新增条目：C:\mingw64\bin。
  4. 依次点击确定保存。
✅ 验证：打开新终端输入 gcc --version，若显示版本信息则配置成功。 
gcc --version
g++ --version

3. Linux / macOS 配置 GCC

大多数 Linux 发行版和 macOS（安装 Xcode Command Line Tools 后）已自带 GCC。如需手动安装： 

# Ubuntu/Debian
sudo apt update
sudo apt install build-essential

# macOS（安装 Homebrew 后）
brew install gcc

安装后检查 /usr/bin/gcc 是否已在 PATH 中：

which gcc
echo $PATH | grep gcc
💡 提示：macOS 上的 gcc 实际上是 Clang 的别名。如需真正 GNU GCC，使用 gcc-12（版本号因安装而异）。

4. Visual Studio MSVC 命令行配置

Visual Studio 不将编译器直接加入系统 PATH，而是通过 Developer Command Prompt 临时设置环境。但你可以手动添加：

  1. 找到 MSVC 编译器目录，例如：
    C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Tools\MSVC\14.40.33807\bin\Hostx64\x64
  2. 将该路径添加到系统 PATH。
  3. 同时添加 INCLUDE 和 LIB 变量指向对应的 include 和 lib 文件夹。
⚠️ 注意：手动配置 MSVC 环境变量较为复杂且易出错，推荐直接使用 Visual Studio 自带的“开发人员命令提示符”。

5. 使用 CMake 时的环境变量

CMake 会自动检测系统中的编译器，你也可以通过环境变量强制指定：

# Linux/macOS
export CC=/usr/bin/gcc
export CXX=/usr/bin/g++

# Windows PowerShell
$env:CC="C:\mingw64\bin\gcc.exe"
$env:CXX="C:\mingw64\bin\g++.exe"
✅ 最佳实践：将上述 export 命令写入 .bashrc 或 .zshrc，实现永久生效。

6. 多版本编译器管理

推荐使用 update-alternatives（Linux）或 Homebrew（macOS）切换 GCC 版本： 

# Linux
sudo update-alternatives --config gcc

# macOS
brew switch gcc 12
