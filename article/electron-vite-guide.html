⚡ Electron-Vite 从入门到打包：完整使用指南 — 2026-06-23

Electron-Vite 是一个新一代的 Electron 开发与构建工具，它结合了 Vite 的极速开发体验和 Electron 的桌面应用能力。本文将带你从零开始，完成项目搭建、开发调试、依赖管理，直至最终打包发布的全流程。

1. 什么是 Electron-Vite？

Electron-Vite 并非一个简单的脚手架，而是一个构建工具。它旨在解决传统 Electron 开发中常见的痛点，如缓慢的热更新、复杂的 Webpack 配置以及繁琐的多进程管理。其核心优势在于：

  • 极速启动：基于 Vite 的 ESM 原生模块加载，开发服务器秒级启动。
  • 智能 HMR：主进程、渲染进程和预加载脚本都支持热更新，无需手动重启。
  • 统一配置：通过一个 electron.vite.config.js 文件即可管理所有进程的构建配置。
  • 内置最佳实践：集成了资源处理、代码分割、安全策略等开箱即用的功能。

2. 环境准备与项目创建

在开始之前，请确保你的电脑上已经安装了 Node.js（推荐 v18 及以上版本）和一个你喜欢的包管理器（npm、yarn 或 pnpm）。

2.1 使用模板创建项目

Electron-Vite 官方提供了一个非常方便的脚手架命令。打开终端，执行以下命令：

# 使用 npm
npm create @quick-start/electron@latest my-electron-app

# 或使用 yarn
yarn create @quick-start/electron my-electron-app

# 或使用 pnpm
pnpm create @quick-start/electron my-electron-app

按照提示选择你喜欢的框架（如 Vue、React 或 Vanilla），然后进入项目目录并安装依赖：

cd my-electron-app
npm install
✅ 最佳实践：推荐使用 pnpm，因为它不仅速度快，还能有效避免幽灵依赖问题。

3. 项目结构与核心配置

创建完成后，你会看到如下项目结构：

my-electron-app/
├── electron/          # 主进程和预加载脚本源码
│   ├── main/
│   │   └── index.js
│   └── preload/
│       └── index.js
├── src/               # 渲染进程源码 (你的前端应用)
│   ├── App.vue
│   └── main.js
├── resources/         # 应用图标等静态资源
├── electron.vite.config.js  # 核心配置文件
└── package.json

3.1 解读配置文件

electron.vite.config.js 是整个构建流程的核心。它继承自 Vite 的配置，并为 Electron 的三个进程分别提供了配置入口：

import { defineConfig } from 'electron-vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  // 主进程构建配置
  main: {
    plugins: [] // 主进程一般不需要太多插件
  },
  // 预加载脚本构建配置
  preload: {
    plugins: []
  },
  // 渲染进程构建配置 (标准 Vite 配置)
  renderer: {
    plugins: [vue()]
  }
})

4. 开发与调试

开发体验是 Electron-Vite 最大的亮点。你只需要运行一个命令：

npm run dev

这个命令会同时做三件事：

  1. 启动 Vite 开发服务器，为渲染进程提供 HMR。
  2. 编译主进程和预加载脚本，并在文件变化时重新编译。
  3. 自动打开 Electron 窗口，并将渲染进程指向本地开发服务器。
💡 提示：当你修改主进程代码后，Electron 窗口会自动刷新，无需手动关闭重开。这是通过 Electron-Vite 内置的进程管理实现的。

5. 引入依赖与 Node.js 模块

在 Electron 开发中，经常需要在主进程中使用 Node.js 原生模块（如 fs, path）或第三方 Node 包。Electron-Vite 对此做了很好的处理：

  • 主进程：默认将 electron 和 Node.js 内置模块标记为外部依赖，不会被打包进 bundle，而是直接通过 require 引用。
  • 渲染进程：默认不暴露 Node.js API，以保证安全性。如果你需要使用，可以在 electron.vite.config.js 的渲染进程配置中开启 nodeIntegration，但更推荐通过预加载脚本暴露。
// electron/preload/index.js
const { contextBridge, ipcRenderer } = require('electron')

contextBridge.exposeInMainWorld('electronAPI', {
  getAppVersion: () => ipcRenderer.invoke('get-app-version')
})

6. 打包与分发

Electron-Vite 集成了 electron-builder，使得打包变得非常简单。你只需要在 package.json 中配置好 builder 选项，然后运行命令即可。

6.1 配置 package.json

在 package.json 中添加 "build" 字段：

{
  "name": "my-electron-app",
  "version": "1.0.0",
  "main": "./out/main/index.js",
  "scripts": {
    "dev": "electron-vite dev",
    "build": "electron-vite build",
    "postinstall": "electron-builder install-app-deps",
    "dist": "electron-vite build && electron-builder"
  },
  "build": {
    "appId": "com.example.myapp",
    "productName": "MyApp",
    "directories": {
      "output": "release"
    },
    "win": {
      "target": ["nsis", "portable"]
    },
    "mac": {
      "target": ["dmg", "zip"]
    },
    "linux": {
      "target": ["AppImage", "deb"]
    }
  }
}

6.2 执行打包

根据不同平台，运行相应的命令：

# 打包当前平台
npm run dist

# 打包特定平台 (需要安装对应平台的依赖)
npm run dist -- --win
npm run dist -- --mac
npm run dist -- --linux
⚠️ 注意：在 macOS 上打包 Windows 应用或在 Windows 上打包 macOS 应用属于交叉编译，通常需要额外配置，且成功率不高。建议在 CI/CD 环境中使用对应平台的原生 Runner 进行打包。

7. 常见问题与优化

7.1 打包体积过大

Electron 应用的体积主要由 Chromium 和 Node.js 决定。你可以通过以下方式优化：

  • 使用 asar 压缩源码（默认启用）。
  • 移除不必要的依赖，尤其是大型的 Node.js 模块。
  • 在 electron-builder 配置中使用 files 字段排除无用文件。

7.2 自动更新

结合 electron-updater 可以实现应用的自动更新。你需要配合一个更新服务器（如 GitHub Releases、S3 或自己搭建的服务）。

const { autoUpdater } = require('electron-updater')

autoUpdater.checkForUpdatesAndNotify()

8. 总结

Electron-Vite 极大地简化了 Electron 的开发流程。从创建项目到打包发布，你只需要掌握几个核心命令和一份简洁的配置文件。希望本文能帮助你顺利上手，构建出优秀的桌面应用。

阶段 关键命令/文件 说明
创建项目 npm create @quick-start/electron 使用官方脚手架初始化
开发调试 npm run dev 启动开发模式，支持 HMR
构建生产 npm run build 编译所有进程的代码
打包分发 npm run dist 生成安装包或便携版
