🚀 Express.js 从入门到部署：全栈后端开发指南 — 2026-06-23

Express.js 是 Node.js 生态中最流行、最轻量的 Web 框架之一。它提供了简洁的路由系统、强大的中间件机制，以及丰富的生态系统。无论你是构建 RESTful API 还是服务端渲染应用，Express 都是绝佳的选择。本文将从项目初始化开始，逐步带你掌握 Express 的核心用法，并最终实现生产级部署。

1. 环境准备与项目初始化

首先确保已安装 Node.js（推荐 v18 LTS 或更高版本）。然后创建一个新目录并初始化项目：

mkdir my-express-app
cd my-express-app
npm init -y

安装 Express：

npm install express

创建入口文件 index.js，编写最基本的服务器：

const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(port, () => {
  console.log(`Server running at http://localhost:${port}`);
});

运行 node index.js，访问 http://localhost:3000 即可看到效果。

✅ 推荐：使用 nodemon 实现自动重启。
npm install --save-dev nodemon，然后在 package.json 中添加脚本："dev": "nodemon index.js"

2. 路由与请求处理

Express 的路由系统非常直观。你可以针对不同的 HTTP 方法和路径定义处理函数：

// GET 请求
app.get('/users', (req, res) => {
  res.json([{ id: 1, name: 'Alice' }, { id: 2, name: 'Bob' }]);
});

// POST 请求
app.post('/users', (req, res) => {
  const newUser = req.body;  // 需要 body-parser 中间件
  res.status(201).json(newUser);
});

// 动态路由参数
app.get('/users/:id', (req, res) => {
  const userId = req.params.id;
  res.send(`获取用户 ${userId} 的信息`);
});

2.1 使用 Router 模块化路由

当路由增多时，建议使用 express.Router() 进行拆分：

// routes/users.js
const router = express.Router();

router.get('/', (req, res) => { /* ... */ });
router.post('/', (req, res) => { /* ... */ });

module.exports = router;

// index.js
const usersRouter = require('./routes/users');
app.use('/users', usersRouter);

3. 中间件：Express 的灵魂

中间件是 Express 的核心概念，它是在请求到达路由处理函数之前执行的函数。你可以用它来解析请求体、记录日志、验证身份等。

// 内置中间件：解析 JSON 请求体
app.use(express.json());

// 自定义中间件：请求日志
app.use((req, res, next) => {
  console.log(`${new Date().toISOString()} | ${req.method} ${req.url}`);
  next();  // 必须调用 next() 才能继续
});

// 第三方中间件示例：cors（跨域）
const cors = require('cors');
app.use(cors());
💡 中间件执行顺序：按注册顺序依次执行。如果某个中间件未调用 next()，则请求会被挂起。通常将通用中间件放在最前面，路由中间件放在后面。

4. 错误处理

Express 提供了专门的错误处理中间件，它有四个参数 (err, req, res, next)：

// 必须在所有路由之后注册
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).json({ error: '服务器内部错误' });
});

你也可以在路由中主动抛出错误：

app.get('/error', (req, res, next) => {
  try {
    throw new Error('出错了！');
  } catch (err) {
    next(err);  // 传递给错误处理中间件
  }
});

5. 连接数据库

以 MongoDB 为例，使用 Mongoose 库：

npm install mongoose
const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/myapp')
  .then(() => console.log('数据库连接成功'))
  .catch(err => console.error('数据库连接失败', err));

// 定义模型
const UserSchema = new mongoose.Schema({
  name: String,
  email: String
});
const User = mongoose.model('User', UserSchema);

// 在路由中使用
app.get('/users', async (req, res) => {
  const users = await User.find();
  res.json(users);
});

6. 测试与调试

推荐使用 Jest + Supertest 进行接口测试：

npm install --save-dev jest supertest
// app.test.js
const request = require('supertest');
const app = require('../app');  // 将 app 导出

describe('GET /', () => {
  it('应返回 Hello World', async () => {
    const res = await request(app).get('/');
    expect(res.statusCode).toBe(200);
    expect(res.text).toBe('Hello World!');
  });
});

7. 生产部署

Express 应用可以部署到多种环境。以下是两种主流方式：

7.1 使用 PM2 进程管理

PM2 是 Node.js 应用的生产级进程管理器，支持负载均衡、自动重启、日志管理等。

npm install -g pm2

# 启动应用（自动启用集群模式）
pm2 start index.js -i max

# 其他常用命令
pm2 list           # 查看进程状态
pm2 logs           # 查看日志
pm2 restart all    # 重启所有进程
pm2 save           # 保存进程列表（开机自启需额外配置）

7.2 使用 Docker 容器化

创建 Dockerfile：

FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
EXPOSE 3000
CMD ["node", "index.js"]

构建并运行：

docker build -t my-express-app .
docker run -d -p 3000:3000 my-express-app
⚠️ 安全建议：在生产环境中务必设置 NODE_ENV=production，并使用 helmet、rate-limiter-flexible 等中间件增强安全性。不要使用 root 用户运行容器。

8. 总结

Express.js 以其灵活性和极低的入门门槛，成为了 Node.js 后端开发的基石。从简单的 API 到复杂的企业级应用，Express 都能胜任。结合 PM2、Docker 等工具，你可以轻松地将应用部署到任何环境。

部署 pm2 start index.js -i max 使用 PM2 守护进程
