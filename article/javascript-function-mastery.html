⚡ JavaScript 函数进阶教程 — 2026-06-28

函数是 JavaScript 的核心构建块之一。掌握函数不仅能让你写出更简洁、可复用的代码，还能深入理解闭包、高阶函数、柯里化等高级概念。本文将从基础出发，逐步带你走进 JS 函数的进阶世界。

1. 函数基础回顾

在 ES6 之前，JavaScript 只有一种函数声明方式——function 关键字。ES6 引入了箭头函数，让函数写法更加简洁。我们先快速回顾一下几种定义方式：

// 函数声明
function sum(a, b) {
  return a + b;
}

// 函数表达式
const multiply = function(a, b) {
  return a * b;
};

// 箭头函数（ES6）
const divide = (a, b) => a / b;
💡 提示：函数声明会被提升（hoisting），而函数表达式不会。箭头函数没有自己的 this 和 arguments，适合用于回调场景。

2. 参数处理：剩余参数与默认值

现代 JS 提供了更灵活的参数处理方式，让函数签名更具表现力。

2.1 默认参数

function greet(name = "朋友", greeting = "你好") {
  return `${greeting}，${name}！`;
}
console.log(greet());           // 你好，朋友！
console.log(greet("小明"));     // 你好，小明！
console.log(greet("小红", "嗨")); // 嗨，小红！

2.2 剩余参数

function logAll(first, ...rest) {
  console.log("第一个参数:", first);
  console.log("其余参数:", rest);
}
logAll(1, 2, 3, 4, 5);
// 输出：
// 第一个参数: 1
// 其余参数: [2, 3, 4, 5]
✅ 最佳实践：剩余参数必须放在参数列表的最后一位，且一个函数只能有一个剩余参数。

3. 高阶函数：函数作为一等公民

在 JavaScript 中，函数可以作为参数传递给另一个函数，也可以作为返回值返回。这种函数被称为高阶函数（Higher-Order Function）。

3.1 数组中的高阶函数

map、filter、reduce 是最常用的三个高阶函数：

const numbers = [1, 2, 3, 4, 5];

// map：对每个元素做变换
const doubled = numbers.map(n => n * 2);
console.log(doubled); // [2, 4, 6, 8, 10]

// filter：筛选满足条件的元素
const evens = numbers.filter(n => n % 2 === 0);
console.log(evens);   // [2, 4]

// reduce：累计归约
const total = numbers.reduce((acc, cur) => acc + cur, 0);
console.log(total);   // 15

3.2 自定义高阶函数

// 创建一个「日志装饰器」
function withLogging(fn) {
  return function(...args) {
    console.log(`调用函数 ${fn.name}，参数:`, args);
    const result = fn(...args);
    console.log(`返回结果:`, result);
    return result;
  };
}

const add = (a, b) => a + b;
const loggedAdd = withLogging(add);
loggedAdd(3, 4);
// 输出：
// 调用函数 add，参数: [3, 4]
// 返回结果: 7
💡 提示：高阶函数是函数式编程的基础，它让代码更加抽象和可组合。

4. 闭包：函数与环境的结合

闭包是指一个函数能够记住并访问其词法作用域中的变量，即使该函数在其词法作用域之外执行。它是 JavaScript 最强大的特性之一。

function createCounter() {
  let count = 0; // 私有变量
  return function() {
    count++;
    return count;
  };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3

// 每次调用都共享同一个 count 变量
const anotherCounter = createCounter();
console.log(anotherCounter()); // 1（新的独立闭包）
⚠️ 注意：闭包会持有外部变量的引用，可能导致内存无法释放。在使用闭包时要注意避免不必要的内存泄漏，尤其是在大型应用中。

4.1 闭包的经典应用：防抖与节流

// 防抖：多次触发只执行最后一次
function debounce(fn, delay = 300) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流：固定时间间隔内只执行一次
function throttle(fn, interval = 300) {
  let lastTime = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}

5. 立即执行函数表达式（IIFE）

IIFE（Immediately Invoked Function Expression）是在定义后立即执行的函数。它常用于创建独立的作用域，避免变量污染全局。

(function() {
  var privateVar = "只有我能访问";
  console.log(privateVar);
})();
// 输出: 只有我能访问
// privateVar 在外部不可访问

// 带参数的 IIFE
const result = (function(a, b) {
  return a * b;
})(5, 6);
console.log(result); // 30
✅ 最佳实践：在现代模块化开发中，IIFE 的使用场景有所减少，但在一些遗留代码或需要隔离作用域的场景下仍然非常有用。

6. 递归：函数调用自身

递归是一种编程技巧，函数内部调用自身来解决问题。经典的例子包括阶乘、斐波那契数列和树的遍历。

// 计算阶乘 n!
function factorial(n) {
  if (n <= 1) return 1;       // 基线条件
  return n * factorial(n - 1); // 递归步骤
}
console.log(factorial(5)); // 120

// 斐波那契数列（优化版：带记忆）
function fibonacci(n, memo = {}) {
  if (n <= 1) return n;
  if (memo[n]) return memo[n];
  memo[n] = fibonacci(n - 1, memo) + fibonacci(n - 2, memo);
  return memo[n];
}
console.log(fibonacci(10)); // 55
⚠️ 注意：递归深度过大可能导致栈溢出（Stack Overflow）。对于深层递归，可以考虑改用循环或尾递归优化（但 JS 引擎对尾递归的支持有限）。

7. 柯里化：将多参数函数转换为链式单参数函数

柯里化（Currying）是把接受多个参数的函数变换成接受单一参数（最初函数的第一个参数）的函数，并且返回接受余下参数的新函数。

// 普通版本
function add(a, b, c) {
  return a + b + c;
}

// 柯里化版本
function curryAdd(a) {
  return function(b) {
    return function(c) {
      return a + b + c;
    };
  };
}
console.log(curryAdd(1)(2)(3)); // 6

// 通用柯里化工具函数
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    } else {
      return function(...nextArgs) {
        return curried.apply(this, args.concat(nextArgs));
      };
    }
  };
}

const curriedSum = curry((a, b, c) => a + b + c);
console.log(curriedSum(1)(2)(3)); // 6
console.log(curriedSum(1, 2)(3)); // 6
💡 提示：柯里化可以让函数更灵活地复用部分参数，在函数式编程库（如 Lodash、Ramda）中被广泛使用。

8. 异步函数：async / await

ES2017 引入的 async / await 让异步代码看起来像同步代码，极大地改善了 Promise 的可读性。

// 模拟一个异步请求
function fetchUser(id) {
  return new Promise(resolve => {
    setTimeout(() => resolve({ id, name: "用户" + id }), 1000);
  });
}

async function loadUsers(ids) {
  const users = [];
  for (const id of ids) {
    const user = await fetchUser(id);
    users.push(user);
  }
  return users;
}

loadUsers([1, 2, 3]).then(users => console.log(users));
// 大约 3 秒后输出：[{id:1, name:"用户1"}, {id:2, name:"用户2"}, {id:3, name:"用户3"}]
✅ 最佳实践：如果多个异步任务互不依赖，应使用 Promise.all 并发执行以提高性能；await 只在需要顺序执行时才使用。

9. 函数进阶对比一览

概念 核心特点 适用场景
高阶函数 接收或返回函数 回调、装饰器、数组方法
闭包 记住外部作用域的变量 私有变量、防抖节流、工厂函数
IIFE 定义后立即执行 隔离作用域、模块模式
递归 函数调用自身 树/图遍历、分治算法
柯里化 多参转单参链式调用 参数复用、函数组合
async/await 异步代码同步化 异步流程控制、API 请求

10. 总结

JavaScript 的函数远不止是一段可重复调用的代码块。从闭包到高阶函数，从柯里化到异步编程，函数在 JS 中扮演着极其灵活的角色。掌握这些进阶用法，能帮助你写出更优雅、更健壮的代码。
