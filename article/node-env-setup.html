🟢 Node.js 环境变量配置与 .env 最佳实践 — 2026-06-15

Node.js 通过 process.env 访问环境变量。合理使用环境变量可以让你的应用更安全、更灵活。

1. 设置 Node.js 相关的环境变量

常用变量

  • NODE_ENV: 指定运行环境（development / production / test）
  • PORT: 应用监听端口
  • DATABASE_URL: 数据库连接字符串

临时设置（命令行）

# Windows (CMD)
set NODE_ENV=production && node app.js

# PowerShell
$env:NODE_ENV="production"; node app.js

# macOS/Linux
NODE_ENV=production node app.js

2. 使用 dotenv 加载 .env 文件

dotenv 是最流行的 npm 包，用于将 .env 文件中的变量加载到 process.env。 

# 安装
npm install dotenv

# .env 文件（位于项目根目录）
PORT=3000
DB_HOST=localhost
DB_USER=admin
DB_PASS=s3cret!

# 在入口文件顶部引入
require('dotenv').config();

console.log(process.env.PORT); // 3000
✅ 最佳实践：创建 .env.example 作为模板提交到 Git，真正的 .env 添加到 .gitignore。

3. 区分不同环境的配置

可以使用多个 .env 文件，如 .env.development、.env.production，并通过 NODE_ENV 决定加载哪个。 

const envFile = `.env.${process.env.NODE_ENV || 'development'}`;
require('dotenv').config({ path: envFile });

4. 在 package.json 中使用 cross-env

为了解决跨平台设置环境变量的差异，推荐使用 cross-env。

# 安装
npm install --save-dev cross-env

# package.json scripts
"scripts": {
  "start": "cross-env NODE_ENV=production node server.js",
  "dev": "cross-env NODE_ENV=development nodemon server.js"
}

5. 安全性提醒

  • 不要在客户端代码中暴露敏感环境变量（Webpack 的 DefinePlugin 需谨慎）。
  • 生产环境中使用密钥管理服务（如 AWS Secrets Manager）替代 .env。
  • 定期轮换密钥。
