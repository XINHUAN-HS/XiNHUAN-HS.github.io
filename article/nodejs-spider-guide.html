🕷️ Node.js 爬虫脚本实战指南 — 从入门到部署 — 2026-06-27

网络爬虫是自动化获取互联网数据的利器。在 Node.js 生态中，凭借其非阻塞 I/O 和丰富的包管理工具，编写高效爬虫变得异常简单。本文将从环境准备讲起，逐步带你实现一个支持 静态页面抓取、动态渲染爬取、反爬绕过以及 数据持久化的完整爬虫脚本，并分享生产环境的部署与监控经验。

1. 技术选型与核心库

在开始之前，我们先梳理一下 Node.js 爬虫常用的依赖库及其适用场景：

库名 用途 适用场景
axios HTTP 客户端，发起网络请求 静态页面、REST API
cheerio 轻量级 HTML 解析，类 jQuery 语法 服务端 DOM 操作
puppeteer 无头浏览器控制 SPA、JS 动态渲染、截图
playwright 跨浏览器自动化（Chromium / Firefox / WebKit） 多浏览器兼容性爬取
node-cron 定时任务调度 周期性增量爬取
winston 结构化日志 日志记录与告警
💡 提示：如果你的目标网站内容由 JavaScript 动态渲染（例如 React / Vue 构建的页面），请优先选择 puppeteer 或 playwright；若只是纯 HTML 静态页面，axios + cheerio 组合足矣，性能更高且资源消耗更小。

3. 实战一：静态页面爬虫（axios + cheerio）

我们以抓取「腾讯云开发者社区」的文章列表为例，演示最经典的 GET → 解析 → 提取流程。

// src/crawlers/example.js
const axios = require('axios');
const cheerio = require('cheerio');
const { logger } = require('../utils/logger');

async function fetchArticleList(url) {
  try {
    const { data } = await axios.get(url, {
      headers: {
        'User-Agent':
          'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 ' +
          '(KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36',
        'Accept-Language': 'zh-CN,zh;q=0.9',
      },
      timeout: 10000,
    });

    const $ = cheerio.load(data);
    const articles = [];

    // 假设每篇文章都在 .article-item 容器中
    $('.article-item').each((i, el) => {
      const title = $(el).find('.article-title a').text().trim();
      const link = $(el).find('.article-title a').attr('href');
      const summary = $(el).find('.article-summary').text().trim();
      const date = $(el).find('.article-date').text().trim();

      if (title) {
        articles.push({ title, link, summary, date });
      }
    });

    logger.info(`成功抓取 ${articles.length} 篇文章`);
    return articles;
  } catch (err) {
    logger.error('抓取失败:', err.message);
    throw err;
  }
}

module.exports = { fetchArticleList };
✅ 最佳实践：始终设置合理的 User-Agent 和 timeout，避免被服务器拒绝连接。对于高频请求，建议加入指数退避重试机制。

4. 实战二：动态渲染爬虫（Puppeteer）

当目标页面需要执行 JavaScript 才能展示真实内容时，Puppeteer 是最可靠的选择。下面是一个抓取 SPA 页面的例子：

// src/crawlers/spa.js
const puppeteer = require('puppeteer');
const { logger } = require('../utils/logger');

async function crawlDynamicPage(url) {
  const browser = await puppeteer.launch({
    headless: true,           // 无头模式
    args: [
      '--no-sandbox',
      '--disable-setuid-sandbox',
      '--disable-dev-shm-usage', // 减少共享内存占用
      '--disable-gpu',
    ],
  });

  try {
    const page = await browser.newPage();
    // 拦截不必要的资源以提升速度
    await page.setRequestInterception(true);
    page.on('request', (req) => {
      if (['image', 'stylesheet', 'font'].includes(req.resourceType())) {
        req.abort();
      } else {
        req.continue();
      }
    });

    await page.goto(url, {
      waitUntil: 'networkidle2', // 等待网络空闲
      timeout: 30000,
    });

    // 等待关键元素出现
    await page.waitForSelector('.post-list', { timeout: 10000 });

    // 执行 DOM 提取
    const data = await page.evaluate(() => {
      const items = document.querySelectorAll('.post-item');
      return Array.from(items).map((item) => ({
        title: item.querySelector('.post-title')?.innerText?.trim(),
        url: item.querySelector('.post-title a')?.href,
        votes: item.querySelector('.vote-count')?.innerText?.trim(),
      }));
    });

    logger.info(`动态页面抓取完成，共 ${data.length} 条记录`);
    return data;
  } finally {
    await browser.close();
  }
}

module.exports = { crawlDynamicPage };
⚠️ 注意：Puppeteer 每次启动浏览器都会消耗较多资源。在生产环境中，建议使用 浏览器池 或复用同一个浏览器实例，避免频繁创建/销毁。

5. 反爬策略与应对方案

现实中的网站往往带有反爬机制，以下是常见挑战与应对措施：

  • IP 频率限制 — 使用代理轮转（Proxy Rotation），搭配 node-fetch 或 axios 的拦截器动态更换 IP。
  • Cookie / Session 校验 — 模拟登录后维持 Cookie 会话，可使用 puppeteer 的 page.setCookie() 注入凭证。
  • JavaScript 挑战（如 CloudFlare） — 使用 puppeteer-extra-plugin-stealth 插件隐藏自动化特征。
  • 验证码 — 接入打码平台（如 2Captcha、超级鹰），或采用 OCR 方案（Tesseract.js）。 
// utils/proxy.js — 简单的代理轮转示例
const proxies = [
  'http://proxy1:8080',
  'http://proxy2:8080',
  'http://proxy3:8080',
];
let idx = 0;

function getNextProxy() {
  const proxy = proxies[idx % proxies.length];
  idx++;
  return proxy;
}

module.exports = { getNextProxy };

6. 数据存储与去重

爬取的数据通常需要落地存储。根据数据量和查询需求，可以选择不同的存储方案：

方案 适合场景 示例库
JSON 文件 小规模、一次性爬取 fs.writeFileSync
SQLite 单机、中等规模 better-sqlite3
PostgreSQL / MySQL 生产级、高并发读写 pg / mysql2
Elasticsearch 全文检索与分析 @elastic/elasticsearch

无论使用哪种存储，去重都是必须考虑的环节。简单做法是用 URL 或内容哈希作为唯一键：

// storage/db.js（SQLite 示例）
const Database = require('better-sqlite3');
const db = new Database('./data/spider.db');

db.exec(`
  CREATE TABLE IF NOT EXISTS articles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    url TEXT UNIQUE NOT NULL,
    title TEXT,
    summary TEXT,
    crawled_at DATETIME DEFAULT CURRENT_TIMESTAMP
  )
`);

function insertIfNotExists(article) {
  const stmt = db.prepare(
    'INSERT OR IGNORE INTO articles (url, title, summary) VALUES (?, ?, ?)'
  );
  const result = stmt.run(article.url, article.title, article.summary);
  return result.changes > 0; // true 表示新增，false 表示重复
}

module.exports = { db, insertIfNotExists };

7. 定时调度与监控

利用 node-cron 可以让爬虫按照指定周期自动运行，结合 winston 记录详细日志便于排查问题。

// src/index.js — 入口文件
const cron = require('node-cron');
const { logger } = require('./utils/logger');
const { fetchArticleList } = require('./crawlers/example');
const { insertIfNotExists } = require('./storage/db');

// 每天凌晨 2:00 执行增量抓取
cron.schedule('0 2 * * *', async () => {
  logger.info('定时爬虫任务启动');
  try {
    const list = await fetchArticleList('https://example.com/articles');
    let newCount = 0;
    for (const article of list) {
      if (insertIfNotExists(article)) {
        newCount++;
      }
    }
    logger.info(`本次新增 ${newCount} 篇，累计去重后共 ${list.length} 篇`);
  } catch (err) {
    logger.error('定时任务执行失败:', err);
    // 可在此触发企业微信 / Slack 告警
  }
});

logger.info('爬虫服务已启动，等待定时任务触发...');
💡 提示：对于生产环境，建议配合 PM2 进行进程守护：pm2 start src/index.js --name spider。 同时使用 winston-daily-rotate-file 按天切割日志，避免磁盘爆满。

8. 部署与运维建议

将爬虫部署到云服务器时，以下几点能显著提升稳定性：

  • 资源限制 — 使用 Docker 容器限定 CPU 和内存上限，防止爬虫意外耗尽主机资源。
  • 健康检查 — 暴露一个简单的 HTTP 端点（如 /health），供外部监控系统探测。
  • 错误重试 — 对网络超时、500 错误等进行最多 3 次重试，并递增延迟。
  • 数据备份 — 定期将 SQLite 或数据库文件备份至对象存储（COS / S3）。
# 使用 Docker 部署示例
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
CMD ["node", "src/index.js"]
“爬虫的本质是模拟人类浏览行为，尊重 robots.txt 和使用条款是底线。合理控制频率，不给目标服务器造成压力，才能让爬虫走得更远。”
