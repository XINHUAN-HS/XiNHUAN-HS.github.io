🐍 Python 环境变量配置指南 — 2026-06-20

环境变量是操作系统级别的变量，用于影响程序的行为。对于 Python 开发者来说，正确配置环境变量可以简化开发流程，避免硬编码敏感信息。

1. 什么是环境变量？

环境变量是一组键值对，存储在操作系统中。Python 可以通过 os.environ 访问它们。常见用途包括：数据库连接字符串、API 密钥、调试模式开关等。

2. 设置 Python 相关环境变量

Windows

  • 打开“系统属性” → “高级” → “环境变量”。
  • 在“系统变量”中点击“新建”，输入变量名 PYTHONPATH，值为你的模块搜索路径（如 C:\my_python_libs）。
  • 若要添加 Python 到 PATH，确保 C:\Python39\Scripts\ 和 C:\Python39\ 在 PATH 中。

macOS / Linux

# 临时设置
export PYTHONPATH="/home/user/my_libs:$PYTHONPATH"

# 永久生效（添加到 ~/.bashrc 或 ~/.zshrc）
echo 'export PYTHONPATH="/home/user/my_libs:$PYTHONPATH"' >> ~/.bashrc
source ~/.bashrc

3. 在 Python 中读取环境变量

import os

db_host = os.getenv('DB_HOST', 'localhost')
api_key = os.environ['API_KEY']  # 若不存在会抛出 KeyError

print(f"数据库主机: {db_host}")

4. 使用 .env 文件管理环境变量

推荐使用 python-dotenv 库加载 .env 文件，避免污染系统环境。 

# 安装
pip install python-dotenv

# .env 文件内容
DB_HOST=localhost
DB_USER=root
DB_PASS=secret123

# 在代码中加载
from dotenv import load_dotenv
load_dotenv()

import os
print(os.getenv('DB_HOST'))
💡 最佳实践：永远不要将敏感信息提交到版本控制中。将 .env 添加到 .gitignore。

5. 常见问题

  • Q: 修改环境变量后为什么没生效？
    A: 需要重启终端或重新加载配置文件（source ~/.bashrc）。
  • Q: PYTHONPATH 和 sys.path 的关系？
    A: PYTHONPATH 会自动添加到 sys.path 中，影响模块导入搜索路径。
