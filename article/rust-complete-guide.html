2. 变量、可变性与常量

Rust 中变量默认不可变（immutable），用 mut 关键字使其可变。常量用 const 声明，且必须标注类型。

fn main() {
    // 不可变变量
    let x = 5;
    // x = 6; // ❌ 编译错误：不能对不可变变量二次赋值

    // 可变变量
    let mut y = 10;
    println!("y = {}", y);
    y = 20;
    println!("y 变为 {}", y);

    // 常量（必须显式类型）
    const MAX_POINTS: u32 = 100_000;
    println!("常量 MAX_POINTS = {}", MAX_POINTS);

    // 变量遮蔽（shadowing）
    let z = "hello";
    let z = z.len();   // 同名新变量，类型变了
    println!("z 的长度 = {}", z);
}
输出 y = 10
y 变为 20
常量 MAX_POINTS = 100000
z 的长度 = 5
✅ 最佳实践：优先使用不可变变量，只有必要时才加 mut。变量遮蔽让你可以在同一作用域复用变量名并改变类型。

4. 函数与控制流

函数使用 fn 关键字，返回值用箭头 -> 指定。控制流包括 if/else、loop、while、for。

/// 返回两个数中较大的（演示函数与 if 表达式）
fn max_of(a: i32, b: i32) -> i32 {
    if a > b { a } else { b }  // if 是表达式，不需要 return
}

fn main() {
    // if 作为表达式赋值
    let condition = true;
    let value = if condition { 100 } else { 0 };
    println!("value = {}", value);

    // loop 循环（可带 break 返回值）
    let mut counter = 0;
    let result = loop {
        counter += 1;
        if counter == 5 {
            break counter * 2; // 跳出并返回 10
        }
    };
    println!("loop 返回: {}", result);

    // while 循环
    let mut n = 3;
    while n > 0 {
        print!("{} ", n);
        n -= 1;
    }
    println!();

    // for 循环（最常用）
    let numbers = [1, 2, 3, 4, 5];
    for num in numbers.iter() {
        print!("{} ", num);
    }
    println!();

    // 调用函数
    println!("最大值: {}", max_of(27, 53));
}
输出 value = 100
loop 返回: 10
3 2 1
1 2 3 4 5
最大值: 53
⚠️ 注意：Rust 中 if 是表达式而非语句，因此可以出现在赋值右侧。但务必保证所有分支类型一致。

5. 所有权与借用

所有权是 Rust 最独特的机制——每个值只有一个所有者，离开作用域即被释放。借用（引用）允许在不转移所有权的情况下访问数据。

fn main() {
    // ---------- 所有权移动 ----------
    let s1 = String::from("hello");
    let s2 = s1;               // s1 的所有权移动到 s2
    // println!("{}", s1);     // ❌ 编译错误：s1 已失效
    println!("s2 = {}", s2);   // ✅

    // ---------- 克隆（深拷贝） ----------
    let s3 = s2.clone();
    println!("s2 = {}, s3 = {}", s2, s3); // 两者都有效

    // ---------- 借用（不可变引用） ----------
    let len = calculate_length(&s3);
    println!("'{}' 的长度 = {}", s3, len); // s3 仍然可用

    // ---------- 可变借用 ----------
    let mut msg = String::from("hello");
    change_msg(&mut msg);
    println!("修改后: {}", msg);
}

fn calculate_length(s: &String) -> usize {
    s.len() // 不持有所有权，只读取
}

fn change_msg(s: &mut String) {
    s.push_str(", world!");
}
输出 s2 = hello
s2 = hello, s3 = hello
'hello' 的长度 = 5
修改后: hello, world!
💡 核心规则：同一时刻只能有一个可变引用，或者多个不可变引用，但不能同时存在可变与不可变引用。这保证了数据竞争在编译期就被消除。

6. 切片类型

切片（slice）是对集合中一段连续元素的引用，不拥有所有权。字符串切片是最常见的用法。

fn first_word(s: &str) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[..i]; // 返回空格前的切片
        }
    }
    &s[..] // 整个字符串
}

fn main() {
    let sentence = String::from("hello rust world");
    let word = first_word(&sentence[..]);
    println!("第一个单词: {}", word);

    // 数组切片
    let arr = [1, 2, 3, 4, 5];
    let slice = &arr[1..4]; // [2, 3, 4]
    println!("数组切片: {:?}", slice);
}
输出 第一个单词: hello
数组切片: [2, 3, 4]
✅ 最佳实践：函数参数中使用 &str 而非 &String，因为前者更通用，可以接受字符串字面量和 String 的切片。

7. 结构体与方法

结构体（struct）用于自定义数据类型。方法通过 impl 块定义，第一个参数总是 self（或其变体）。

#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

// 方法块
impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }

    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width >= other.width && self.height >= other.height
    }

    // 关联函数（不是方法，类似静态构造器）
    fn square(size: u32) -> Rectangle {
        Rectangle { width: size, height: size }
    }
}

fn main() {
    let rect1 = Rectangle { width: 30, height: 50 };
    let rect2 = Rectangle::square(20);

    println!("rect1 面积: {}", rect1.area());
    println!("rect1 能容纳 rect2? {}", rect1.can_hold(&rect2));
    println!("rect2: {:?}", rect2);
}
输出 rect1 面积: 1500
rect1 能容纳 rect2? true
rect2: Rectangle { width: 20, height: 20 }
💡 提示：#[derive(Debug)] 让结构体可以自动实现调试打印（{:?}）。关联函数用 :: 调用，如 Rectangle::square(20)。

8. 枚举与模式匹配

枚举（enum）可以携带不同类型的数据。match 是 Rust 中极其强大的模式匹配工具，必须穷尽所有分支。

enum IpAddr {
    V4(u8, u8, u8, u8),
    V6(String),
}

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => {
            println!("幸运便士！");
            1
        }
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}

fn main() {
    let home = IpAddr::V4(127, 0, 0, 1);
    let loopback = IpAddr::V6(String::from("::1"));

    // 使用 match 处理枚举
    let coin = Coin::Dime;
    println!("价值 {} 美分", value_in_cents(coin));

    // if let —— 只关心单个模式的简洁写法
    let config_max = Some(3u8);
    if let Some(max) = config_max {
        println!("最大值为 {}", max);
    }
}
输出 价值 10 美分
最大值为 3
⚠️ 注意：match 必须覆盖所有可能性。如果不想处理某些分支，可以用 _ => () 通配符。而 if let 适合只匹配一种情况的场景。

9. 错误处理

Rust 没有异常机制，而是使用 Result<T, E> 和 panic!。可恢复错误用 Result，不可恢复用 panic!。

use std::fs::File;
use std::io::{self, Read};

/// 读取文件内容（演示 ? 运算符传播错误）
fn read_file_content(path: &str) -> Result {
    let mut file = File::open(path)?;  // 如果出错，提前返回 Err
    let mut contents = String::new();
    file.read_to_string(&mut contents)?;
    Ok(contents)
}

fn main() {
    // 故意打开不存在的文件
    let result = read_file_content("nonexistent.txt");

    match result {
        Ok(text) => println!("文件内容:\n{}", text),
        Err(e) => eprintln!("❌ 读取失败: {}", e),
    }

    // unwrap / expect —— 简化但不推荐在生产中滥用
    // let data = read_file_content("Cargo.toml").expect("无法读取 Cargo.toml");
    // println!("{}", data);
}
输出（示例） ❌ 读取失败: No such file or directory (os error 2)
✅ 最佳实践：在库函数中优先返回 Result，让调用者决定如何处理错误。使用 ? 运算符可以大幅简化错误传播代码。

10. 泛型与 trait

泛型让代码适用于多种类型；trait 定义了共享行为，类似于其他语言的接口。

// 泛型函数：找出数组中最大的元素（需要 PartialOrd 约束）
fn largest(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list.iter() {
        if item > largest {
            largest = item;
        }
    }
    largest
}

// 定义一个 trait
trait Summary {
    fn summarize(&self) -> String;
}

struct Article {
    title: String,
    author: String,
}

impl Summary for Article {
    fn summarize(&self) -> String {
        format!("《{}》by {}", self.title, self.author)
    }
}

fn main() {
    // 泛型函数测试
    let numbers = vec![34, 67, 12, 89, 23];
    println!("最大数字: {}", largest(&numbers));

    let chars = ['y', 'm', 'a', 'q'];
    println!("最大字符: {}", largest(&chars));

    // trait 使用
    let art = Article {
        title: String::from("Rust 入门"),
        author: String::from("心幻"),
    };
    println!("摘要: {}", art.summarize());

    // 也可以写成泛型函数接收实现了 Summary 的类型
    notify(&art);
}

fn notify(item: &impl Summary) {
    println!("通知: {}", item.summarize());
}
输出 最大数字: 89
最大字符: y
摘要: 《Rust 入门》by 心幻
通知: 摘要: 《Rust 入门》by 心幻

11. 常用集合

Vec<T>（动态数组）、HashMap<K, V>（哈希表）和 String 是日常开发中最常用的三种集合。

use std::collections::HashMap;

fn main() {
    // ----- Vec -----
    let mut v: Vec = Vec::new();
    v.push(10);
    v.push(20);
    v.push(30);
    println!("v = {:?}", v);

    // 宏创建
    let v2 = vec![1, 2, 3, 4, 5];
    let third = v2.get(2); // Option<&i32>
    match third {
        Some(val) => println!("第三个元素: {}", val),
        None => println!("索引越界"),
    }

    // 遍历
    for i in &v2 {
        print!("{} ", i);
    }
    println!();

    // ----- HashMap -----
    let mut scores = HashMap::new();
    scores.insert(String::from("Alice"), 95);
    scores.insert(String::from("Bob"), 82);
    scores.insert(String::from("Charlie"), 91);

    let name = String::from("Bob");
    if let Some(score) = scores.get(&name) {
        println!("{} 的成绩: {}", name, score);
    }

    // 遍历
    for (key, value) in &scores {
        println!("{} → {}", key, value);
    }

    // ----- String -----
    let mut s = String::from("你好");
    s.push_str("，世界");
    s.push('！');
    println!("{}", s);
    println!("长度: {}", s.chars().count()); // 实际字符数
}
输出 v = [10, 20, 30]
第三个元素: 3
1 2 3 4 5
Bob 的成绩: 82
Alice → 95
Bob → 82
Charlie → 91
你好，世界！
长度: 6
💡 提示：访问 Vec 元素推荐用 .get(index) 返回 Option，避免直接下标访问导致 panic。HashMap 的 entry API 可以优雅地处理插入或更新逻辑。

12. 生命周期入门

生命周期注解告诉编译器引用之间的存活关系，确保不会产生悬垂引用。这是 Rust 最难的概念之一，但日常使用中多数时候编译器会自动推断。

/// 返回两个字符串切片中较长的那个（需要生命周期标注）
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

fn main() {
    let string1 = String::from("long long string");
    let result;
    {
        let string2 = "short";
        result = longest(string1.as_str(), string2);
        println!("较长的是: {}", result);
    } // string2 在这里结束，但 result 的生命周期与 string1 和 string2 中较短的一致
      // 下面这行如果取消注释会编译错误，因为 string2 已失效
      // println!("{}", result);
}

输出 较长的是: long long string
⚠️ 注意：生命周期注解 'a 并不改变引用的实际存活时间，只是建立约束。多数情况下，函数签名中的生命周期可以省略（遵循 elision 规则）。

13. 闭包与迭代器

闭包是可以捕获环境的匿名函数；迭代器提供了一种高效、链式的集合处理方式。二者结合非常强大。

fn main() {
    // ----- 闭包 -----
    let factor = 3;
    let multiply = |x| x * factor; // 捕获外部变量 factor
    println!("5 * 3 = {}", multiply(5));

    // 闭包作为参数
    let numbers = vec![1, 2, 3, 4, 5, 6];
    let evens: Vec<_> = numbers
        .iter()
        .filter(|&&x| x % 2 == 0)   // 闭包：过滤偶数
        .map(|x| x * 10)           // 闭包：每个乘以 10
        .collect();
    println!("偶数翻十倍: {:?}", evens);

    // ----- 迭代器适配器 -----
    let sum: i32 = (1..=100).sum(); // 1 到 100 求和
    println!("1+2+...+100 = {}", sum);

    // 链式操作：找出所有能被 3 整除的数，取前 5 个，平方后收集
    let result: Vec<_> = (1..)
        .filter(|x| x % 3 == 0)
        .take(5)
        .map(|x| x * x)
        .collect();
    println!("前 5 个 3 的倍数的平方: {:?}", result);
}
输出 5 * 3 = 15
偶数翻十倍: [20, 40, 60]
1+2+...+100 = 5050
前 5 个 3 的倍数的平方: [9, 36, 81, 144, 225]
✅ 最佳实践：优先使用迭代器和闭包组合来处理集合数据，代码更简洁、更具表现力，且性能通常不低于手写循环（因为迭代器是零开销抽象的典范）。

14. 模块系统

Rust 使用 mod、use 和 pub 组织代码。每个 crate 都有一个根模块（main.rs 或 lib.rs）。

// 假设 src/lib.rs 或 src/main.rs 中包含以下内容

/// 公共函数：对外可见
pub fn greet(name: &str) -> String {
    format!("你好，{}！", name)
}

/// 内部辅助函数：仅当前 crate 可见
fn internal_helper() {
    println!("这是一个内部函数");
}

// 子模块
pub mod math {
    /// 加法（公开）
    pub fn add(a: i32, b: i32) -> i32 {
        a + b
    }

    /// 乘法（公开）
    pub fn mul(a: i32, b: i32) -> i32 {
        a * b
    }

    /// 私有函数：仅 math 模块内部可用
    fn private_debug() {
        println!("math 模块内部");
    }
}

fn main() {
    // 使用 use 导入
    use math::{add, mul};
    println!("3 + 5 = {}", add(3, 5));
    println!("4 * 7 = {}", mul(4, 7));
    println!("{}", greet("Rustacean"));
}
输出 3 + 5 = 8
4 * 7 = 28
你好，Rustacean！
💡 提示：Rust 2018 edition 之后，模块路径采用绝对路径（crate::）或相对路径（self::, super::）。pub(crate) 可将可见性限制在当前 crate 内。

15. 并发基础

Rust 通过 std::thread 和消息传递（channel）提供安全并发。所有权和类型系统在编译期杜绝数据竞争。

use std::sync::mpsc;
use std::thread;
use std::time::Duration;

fn main() {
    // ----- 创建线程 -----
    let handle = thread::spawn(|| {
        for i in 1..5 {
            println!("👾 子线程: {}", i);
            thread::sleep(Duration::from_millis(300));
        }
    });

    for i in 1..3 {
        println!("👤 主线程: {}", i);
        thread::sleep(Duration::from_millis(400));
    }

    handle.join().unwrap(); // 等待子线程结束
    println!("--- 线程已汇合 ---");

    // ----- 通道（Channel）消息传递 -----
    let (tx, rx) = mpsc::channel();

    let tx_clone = tx.clone();
    thread::spawn(move || {
        let vals = vec!["ping", "pong", "done"];
        for val in vals {
            tx.send(val).unwrap();
            thread::sleep(Duration::from_millis(200));
        }
    });

    thread::spawn(move || {
        let vals = vec!["hello", "world"];
        for val in vals {
            tx_clone.send(val).unwrap();
            thread::sleep(Duration::from_millis(350));
        }
    });

    // 主线程接收所有消息
    for received in rx {
        println!("📩 收到: {}", received);
    }
}
输出（顺序因调度而异） 👤 主线程: 1
👾 子线程: 1
👾 子线程: 2
👤 主线程: 2
👾 子线程: 3
👾 子线程: 4
--- 线程已汇合 ---
📩 收到: ping
📩 收到: hello
📩 收到: pong
📩 收到: world
📩 收到: done
⚠️ 注意：线程间传递所有权时，move 关键字将捕获的变量移动到闭包中。通道的 send 也会转移所有权，从而保证线程安全。
