🟢 Vue 2 & Vue 3 完整指南：从入门到项目实战 — 2026-06-21

Vue.js 是目前最受欢迎的前端框架之一，广泛应用于单页应用和企业级项目。Vue 2 奠定了生态基础，而 Vue 3 带来了 Composition API、TypeScript 深度集成、性能大幅提升等重大改进。本文将对两个版本进行系统梳理，帮助开发者理解差异、平滑迁移，并掌握实际项目中的核心技能。

1. Vue 2 vs Vue 3：核心区别一览

在深入学习之前，先了解两个版本的主要差异：

维度 Vue 2 Vue 3
响应式原理 Object.defineProperty（无法检测新增/删除属性） Proxy（完美拦截对象所有操作）
API 风格 Options API（data、methods、computed 等选项） Options API + Composition API（setup 函数）
TypeScript 支持 需额外装饰器或类组件 原生支持，类型推导完善
虚拟 DOM 传统 diff 静态标记 + PatchFlag，性能提升 1.3~2 倍
Tree Shaking 不支持 支持，未使用的 API 不打包
Fragment / Teleport / Suspense 不支持 原生支持
💡 版本现状：Vue 2 已于 2023 年 12 月 31 日停止维护（EOL），官方强烈建议新项目使用 Vue 3。但大量存量项目仍运行在 Vue 2 上，因此了解两者都很有必要。

2. 项目创建与初始化

2.1 Vue 2 项目

使用 Vue CLI 创建：

npm install -g @vue/cli
vue create my-vue2-app
# 选择 Vue 2 预设

2.2 Vue 3 项目

推荐使用 Vite 创建（更快）：

npm create vite@latest my-vue3-app -- --template vue
cd my-vue3-app
npm install

也可以使用 Vue CLI（选择 Vue 3 预设）。

3. Options API（Vue 2 / Vue 3 通用）

Options API 是两个版本都支持的写法，适合简单组件：

<template>
  <div>
    <p>{{ message }}</p>
    <button @click="reverseMessage">反转</button>
  </div>
</template>

<script>
export default {
  data() {
    return { message: 'Hello Vue!' }
  },
  methods: {
    reverseMessage() {
      this.message = this.message.split('').reverse().join('')
    }
  }
}
</script>

4. Composition API（Vue 3 特有）

Composition API 通过 setup 函数组织逻辑，更适合复杂组件和逻辑复用： 

<script setup>
import { ref, computed } from 'vue'

const message = ref('Hello Vue 3!')
const reversedMessage = computed(() =>
  message.value.split('').reverse().join('')
)

function reverseMessage() {
  message.value = reversedMessage.value
}
</script>

<template>
  <p>{{ message }}</p>
  <button @click="reverseMessage">反转</button>
</template>
✅ 最佳实践：Vue 3 项目优先使用 <script setup> 语法糖，代码更简洁，类型推导更好。

5. 生命周期钩子对比

Vue 3 的生命周期钩子命名改为 onXxx 形式，且只能在 setup 中调用：

Vue 2 Vue 3 (Composition API) 说明
beforeCreate —（在 setup 中直接执行） 实例初始化前
created —（在 setup 中直接执行） 实例创建完成
beforeMount onBeforeMount 挂载前
mounted onMounted 挂载后
beforeUpdate onBeforeUpdate 更新前
updated onUpdated 更新后
beforeDestroy onBeforeUnmount 卸载前
destroyed onUnmounted 卸载后

6. 从 Vue 2 迁移到 Vue 3

如果你正在维护 Vue 2 项目，可以考虑逐步迁移。主要注意事项：

  • 破坏性变更：$listeners 合并到 $attrs，v-model 默认使用 modelValue，key 不再支持 v-if 分支等。
  • 全局 API 更改：Vue.prototype → app.config.globalProperties，Vue.component → app.component。
  • 过滤器 filters 已被移除，改用计算属性或方法。
  • 事件总线：推荐使用 mitt 或 Pinia 替代 $on/$emit。

官方提供了 Migration Build（兼容构建版本），可以让你在 Vue 3 项目中运行大部分 Vue 2 代码，逐步替换。

⚠️ 注意：不建议直接在同一个项目中混合使用 Options API 和 Composition API，保持风格一致更利于维护。迁移时可先用 Options API 过渡，再逐步重构。

7. 状态管理：Vuex vs Pinia

Vue 2 时代的标准方案是 Vuex，Vue 3 官方推荐使用 Pinia：

特性 Vuex 4 (Vue 3) Pinia
TypeScript 支持 一般 优秀，类型推断完整
DevTools 支持 支持，体验更好
模块化 嵌套 modules 扁平 store，天然模块化
Mutation 必需 无 mutation，直接修改 state
体积 较大 约 1KB 
// Pinia 示例
import { defineStore } from 'pinia'

export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 }),
  actions: {
    increment() { this.count++ }
  }
})

8. 路由：Vue Router 3 vs 4

Vue 2 对应 Vue Router 3，Vue 3 对应 Vue Router 4。主要变化：

  • 创建方式：new VueRouter() → createRouter()
  • 历史模式：mode: 'history' → history: createWebHistory()
  • 导航守卫：next() 参数变化，推荐返回 false 或路由地址。

9. 生产构建与部署

Vue 2 和 Vue 3 的构建命令相同：

npm run build

输出目录默认为 dist，可直接部署到 Nginx、Vercel、Netlify 等静态服务器。对于 SPA 路由，记得配置 fallback 规则： 

# Nginx 配置
location / {
    try_files $uri $uri/ /index.html;
}
“Vue 3 不仅仅是性能的提升，更是开发体验的一次革命。Composition API 让逻辑复用变得前所未有的优雅。” —— Evan You

10. 总结

无论你选择 Vue 2 还是 Vue 3，Vue 的核心思想——声明式渲染、组件化、渐进式框架——始终不变。对于新项目，毫不犹豫地选择 Vue 3 + Vite + Pinia 组合；对于旧项目，可以利用 Migration Build 平稳过渡。掌握两个版本的异同，将使你在不同项目中游刃有余。

场景 推荐技术栈 备注
全新项目 Vue 3 + Vite + Pinia + Vue Router 4 官方推荐，性能最佳
维护 Vue 2 项目 Vue 2 + Vue CLI + Vuex 3 稳定，但需规划迁移
迁移过渡期 Vue 3 + Migration Build 兼容 Vue 2 代码，逐步重构
